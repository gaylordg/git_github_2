Gérer son code avec Git et GitHub

commit, git log et branche

1 - Qu'est ce qu'un commit ?

Un commit, c'est un marqueur de toutes les opérations effectuées sur son code. Ainsi, l'ajout, la suppression ou la modification d'un ou plusieurs fichiers peut être conservé en une version tout en expliquant les raisons de ces changements grâce aux messages.

Commande :

git commit -m « un message» Permet de faire un commit en laissant un message expliquant les modifications apportées

2 - A quoi sert la commande git log ?

La commande permet de voir l'historique de tous les commits. Le dernier commit effectué se trouvera en premier.
Pour chaque commit on retrouvera :

  • Le SHA : l'identifiant unique du commit
  • le nom de l'auteur
  • la date
  • le message : Il est très important de mettre un message clair pour visualiser les raisons des modifications

3 – Qu'est qu'une branche ?

Pour travailler sur une nouvelle fonctionnalité ou expérimenter une légère modification d'un projet sans interférer sur le code principal ou branche principale, on se détache de celui-ci en créant une branche secondaire, la finalité étant de fusionner ces deux branches. Par convention la branche principale se nomme « master ».

Quelques commandes :

git branch nom_branche Permet de créer une branche
git checkout nom branche Permet de se positionner sur une branche
git merge nom_branche Permet de fusionner une branche avec celle sur laquelle on est positionnné
